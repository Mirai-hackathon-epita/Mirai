# Global Plan & Feature Set — Adaptive Learning Platform (POC / Demo)

> Scope: this is the **demo POC** feature set, not a shipping product. Depth over breadth.
> Theme to win: **Agentic AI & Autonomy**. Every feature below is justified against the judging grid (see §8).

---

## 1. Product in one sentence

An adaptive learning platform where **each student is paired with their own autonomous AI tutor** that plans, generates, grades, and adapts exercises **at that student's own pace** — while the teacher supervises and intervenes through a live dashboard.

---

## 2. Core principle — one autonomous agent per student, self-paced

This is the spine of the product and the heart of the theme.

- Each student has a **dedicated tutor agent** running its own loop:
  **plan** (decide what this student needs next, using the skill graph + memory)
  → **act** (generate exercise / grade the photo / explain a concept / update memory)
  → **reflect** (diagnose the error, adjust difficulty, change representation)
  → **escalate** (flag the teacher only when it matters)
  → repeat.
- **Agents are independent and self-paced.** No global batch, no student ever waits on another. Everyone advances at their own speed — this is the product's core promise.
- Each agent works **proactively for its own student**: as soon as a student finishes, its agent grades, updates the graph, diagnoses, and **pre-generates the next exercise(s)** so there is never a wait.
- **Autonomy proof for judges:** many independent agents, each pursuing one learner's mastery goal with minimal human input.

---

## 3. Teacher features

- **Course upload.** Teacher drops a math course (one topic end-to-end for the demo, e.g. derivatives).
- **AI course enrichment.** The agent ingests the material and:
  - extracts a **concept prerequisite graph** (the structure that powers all planning),
  - generates **interactive visualizations** for key concepts,
  - enriches the course into an interactive experience.
- **Review & publish.** Teacher reviews the enriched course and publishes it to the class.
- **Set a mastery target (optional).** The teacher can set a deadline ("the class masters derivatives by the TD on June 23"). Each student's agent then plans backward from it (see §5.8).
- **Live teacher dashboard:**
  - per-student **skill graphs** (mastery + gaps),
  - **escalations** from student agents, each carrying a **diagnosis + a suggested short intervention** ("Léa is stuck on factoring; recommend a 5-min review of common factors"),
  - **class-level auto-insight** synthesized across students ("60% of the class is blocked on the chain rule — re-teach it first in TD"). This is a read-only synthesis over the independent agents, never a gate on students.
- **Ask the AI about progress.** The teacher can ask, in natural language, about overall class progress or a specific student ("How is Léa doing this week?", "Who should I pull aside in TD?", "Where is the class blocked?"). The assistant answers from the agents' persistent memory and skill graphs.

---

## 4. Student features

- **Interactive course view** — the enriched course with generated visualizations.
- **Grounded chat tutor** — chatbot grounded in the course (RAG) *and* aware of the student's current gaps. It tutors toward mastery, it does not just chat.
- **Personalized exercises** — generated by the planning step, calibrated to this student's level and profile (not random).
- **Photo submission + OCR grading** — student writes the solution on paper, photographs it; the agent reads the handwriting and grades **step by step**, pinpointing the failing step (not just right/wrong).
- **Multi-modal learning** (see §5) — any explanation available as video / audio / written.
- **Self-paced progression** — the student's own agent always has the next step ready.

---

## 5. Cross-cutting capabilities

### 5.1 Concept prerequisite graph (DAG)
Not a flat list of mastery %. A graph of concepts with **prerequisite edges**. The agent plans over it: if a student fails "integration by parts," the agent checks the prerequisite "derivatives" and, if weak, drills *that* first.

### 5.2 Persistent per-student memory
Every agent remembers the student's full history: attempts, errors, mastery per concept, inferred cognitive profile, past diagnoses. This memory drives incremental difficulty and continuity across sessions.

### 5.3 Misconception diagnosis loop ⭐ (DEMO CENTERPIECE)
The agent does not just mark an answer wrong. It:
1. forms a **hypothesis** about the underlying misconception (e.g. "the student forgets the coefficient in the power rule"),
2. **generates a probe exercise** designed to test that hypothesis,
3. **confirms or refutes** it from the student's response,
4. applies a **targeted fix** (scaffolded exercise on exactly the broken sub-skill) and updates memory.

This hypothesis → experiment → conclusion behavior is the strongest innovation and agentic signal — it is the feature we polish most and build the narrative around.

### 5.4 Multi-modal learning (cognitive profile adaptation)
Any explanation — in the chat tutor **or** when reviewing the course — can be delivered three ways:
- **Short video** — for visual learners (animation / visualization clip),
- **Audio narration** — for auditory learners,
- **Write-down text** — for written learners (dyslexia-friendly formatting included).

The default modality is chosen from the student's **declared or inferred** profile and is **switchable on demand**. The agent can **infer** a profile shift (e.g. the student keeps failing text-heavy problems but succeeds on visual ones) and adapt representation on its own.

### 5.5 Agent reasoning feed (observability)
A live panel surfacing each agent's autonomous decisions in plain language:
*"Léa failed 2 fraction problems → hypothesis: confuses numerator/denominator rules → generated a probe → confirmed → lowered difficulty, generated a scaffolded exercise, flagged teacher."*
Autonomy that judges cannot see cannot be scored — this panel makes it visible. Cheap to build, high payoff for demo + theme.

### 5.6 Calibration & mastery model
- **Open placement diagnostic.** A brand-new student starts with an **open, free-form conversation** about the topic (an open speech window). The agent assesses the student's understanding from this free speech and seeds an **initial belief** over the concept graph (a prior).
- **Continuous re-assessment.** That belief is **revised by evidence** as the student completes exercises — the prior is never final; every exercise outcome updates it.
- **Completion = mastery threshold.** A concept turns "mastered" (green) only when the student crosses a **mastery threshold** (not a single correct answer); the agent then unlocks and moves on to dependent concepts in the graph.

### 5.7 Shared diagnostic memory
Each agent logs the misconceptions it **confirms** (via §5.3) into a memory shared across all student agents, keyed by concept. It acts as a **shared, growing prior**: when a new student makes a familiar mistake, their agent matches it against known misconceptions and goes straight to the proven fix, instead of re-running the full hypothesis → probe loop from scratch.
- **Faster, cheaper diagnosis:** matching a known misconception skips expensive generation/probing steps, reducing LLM calls per diagnosis. The more the class is used, the warmer the prior and the better the cold-start for new students.
- **Honest framing:** this is a shared, growing knowledge store (a warm prior over misconceptions), **not** "emergent behavior." We claim cost, speed, and better cold-start — not emergence.
- Still **fully self-paced**: the memory is a read/write store, never a batch that blocks any student.

### 5.8 Goal-directed planning toward a deadline
When the teacher sets a mastery target (§3), each student's agent **plans backward** from the deadline: it estimates the concepts still to master, the time remaining, and **adjusts intensity per learner** (more exercises, tighter difficulty steps, earlier escalation) to stay on track.
- Upgrades autonomy from "reacts well to input" to **"achieves a complex, multi-step goal"** — the wording that separates a 4 from a 5 on the autonomy criterion.
- Per-student and independent: each agent owns its own plan toward the shared deadline; no student waits on another.

## 6. Demo narrative (~2 min, the spine)

1. **Teacher** uploads a derivatives course → agent extracts the concept graph + generates an interactive visualization → teacher reviews and publishes.
2. **Student A** (visual learner, has gaps): sees the enriched interactive course, switches an explanation to **video** mode, chats with the tutor, gets a calibrated exercise, **writes it on paper and photographs it**. Agent OCRs → grades step by step → finds the error in step 2 → **runs the misconception diagnosis loop** (hypothesis → probe → confirm) → updates the skill graph → generates a scaffolded follow-up → **flags the teacher**. All narrated in the agent reasoning feed.
3. **Student B** (auditory learner, further along): show that B's *independent* agent generated a **different modality (audio) and a harder exercise**, and that B is at a **different point in the course** — proving self-paced, per-student autonomy with no waiting.
4. **Teacher dashboard:** live skill graphs for both students, Student A's escalation with its suggested intervention, and the **class-level insight**.

## 7. Out of scope for the POC

- Real auth / multiple classes → **seed 3 students** with distinct profiles + histories.
- Broad subject coverage → **one math topic** end-to-end (derivatives).
- Production-grade content moderation, billing, mobile apps, etc.

## 9. Build priority — ranked for the demo

Ranked by **criticality + usefulness to win** (synthesis of the jury review). Build top-down. If time runs short, **cut from the bottom**: Tier 4 is deferred by default; trim Tier 3 to seeded/pre-rendered showpieces; **never cut Tier 0–1**. The critical path of the live demo must stay inside Tier 0–2 — every Tier 3 item stays *off* the critical path (one-click, seeded, or pre-rendered).

| # | Feature (§) | Why this rank — win leverage | Build treatment |
|---|---|---|---|
| 1 | Persistent per-student memory — Redis (§5.2) | Backbone for all state; cleanest sponsor-stack win; cheap. | Build real (Redis hashes/sorted sets). |
